Module 1

JavaScript Fundamentals

Back to contents

React becomes much easier once variables, functions, arrays, objects, and references feel natural.

Core ideas
  • Prefer const by default and use let only when reassignment is required.
  • Use arrow functions for callbacks and compact reusable logic.
  • Learn destructuring, spread, and rest because they appear everywhere in React state updates.
  • Practice map, filter, and reduce so rendering lists feels easy.
  • Understand that arrays and objects are copied by reference unless you create a new copy.
Common mistake: mutating an object directly and expecting React to notice a change.
Practice checklist
  • Rewrite a normal function as an arrow function.
  • Use destructuring in a function parameter.
  • Copy an array with spread before adding an item.
  • Transform an array with map.
Example code
const users = [
    { id: 1, name: 'Khwaab', score: 82 },
    { id: 2, name: 'Sam', score: 95 }
  ];

  const addBonus = (list, bonus) =>
    list.map(({ score, ...user }) => ({
      ...user,
      score: score + bonus
    }));

  const updatedUsers = addBonus(users, 5);
  // [{ id: 1, name: 'Khwaab', score: 87 }, ...]
Module 2

TypeScript Basics

Back to contents

TypeScript helps you catch mistakes earlier and makes large React apps easier to refactor safely.

What to learn
  • Primitive types, arrays, objects, and function return types.
  • type and interface for describing data shapes.
  • Union types for values that can be one of several options.
  • Typed component props for safer React components.
Watch out for
  • Do not use any everywhere.
  • Type function parameters and return values.
  • Keep prop names and types consistent.
Example code
type User = {
    id: number;
    name: string;
    role: 'admin' | 'member';
  };

  type UserBadgeProps = {
    user: User;
  };

  function UserBadge({ user }: UserBadgeProps) {
    return <span>{user.name} ({user.role})</span>;
  }
Module 3

React Basics

Back to contents

This is where JavaScript turns into UI: components, JSX, props, state, events, and small reusable pieces.

Essential ideas
  • Modern projects are commonly created with Vite.
  • Components return JSX, which looks like HTML inside JavaScript.
  • Props pass data in; state stores changing data.
  • Event handlers such as onClick and onChange keep the UI interactive.
  • Controlled inputs make forms predictable.
Build this first
  • A greeting component with props.
  • A counter with state and a button click.
  • A text input that updates live on screen.
Example code
import { useState } from 'react';

  function WelcomeCard({ name }) {
    const [likes, setLikes] = useState(0);

    return (
      <section className="card">
        <h2>Hello, {name}</h2>
        <p>Likes: {likes}</p>
        <button onClick={() => setLikes(likes + 1)}>Like</button>
      </section>
    );
  }
Module 4

Debugging

Back to contents

Debugging is a skill, not a last resort. The goal is to find the smallest place where expected behavior diverges from actual behavior.

Debugging workflow
  • Read the full error message, especially the file name and line number.
  • Reproduce the issue with the smallest possible input.
  • Use console.log, browser DevTools, and React Developer Tools.
  • Check props, state, and network responses before guessing.
Useful checks
  • Is a value undefined before it loads?
  • Did you mutate state instead of replacing it?
  • Is a component rendering more often than expected?
Example code
function Profile({ user }) {
    console.log('Profile received:', user);

    if (!user) {
      return <p>Loading user...</p>;
    }

    return <h2>{user.name}</h2>;
  }
Tip: guarding against missing data first often removes “Cannot read properties of undefined” errors.
Module 5

Styling Components

Back to contents

Start with a simple styling approach, then choose utility classes or component-scoped styles when the project grows.

Common options
  • Plain CSS for small projects and quick experiments.
  • CSS Modules for local component styles.
  • Tailwind CSS for utility-first styling.
  • styled-components for CSS-in-JS projects.
Remember
  • Use conditional classes for active or disabled states.
  • Keep style naming consistent.
  • Do not overcomplicate styling for beginner apps.
Example code
function StatusBadge({ isActive }) {
    const className = isActive
      ? 'badge badge--active'
      : 'badge badge--inactive';

    return <span className={className}>
      {isActive ? 'Active' : 'Inactive'}
    </span>;
  }
Module 6

Component Deep Dive

Back to contents

As apps grow, good component structure matters just as much as writing JSX that works.

What to understand
  • Split large screens into smaller focused components.
  • Prefer functional components and hooks in modern React.
  • Use Fragment when you need multiple sibling elements without an extra wrapper.
  • Use refs for DOM access and focus management.
  • Use React.memo only when repeated re-renders are actually a problem.
Component habits
  • Keep components small and purpose-specific.
  • Move repeated logic into custom hooks.
  • Name components after what they render.
Example code
import { useEffect, useRef } from 'react';

  function SearchInput() {
    const inputRef = useRef(null);

    useEffect(() => {
      inputRef.current?.focus();
    }, []);

    return (
      <>
        <label htmlFor="search">Search</label>
        <input id="search" ref={inputRef} />
      </>
    );
  }
Module 7

HTTP Requests & API Handling

Back to contents

Fetching data is one of the first places where real apps feel different from toy examples.

Key ideas
  • Use GET to fetch, POST to create, PATCH/PUT to update, and DELETE to remove.
  • Store loading, data, and error state explicitly.
  • Transform API data before rendering if the UI needs a different shape.
  • Avoid infinite loops in useEffect dependencies.
  • An axios instance can centralize a base URL and shared headers.
Checklist
  • Show a loading state.
  • Show an error state.
  • Use keys when rendering API lists.
  • Keep fetch logic predictable.
Example code
import { useEffect, useState } from 'react';

  function PostsList() {
    const [posts, setPosts] = useState([]);
    const [status, setStatus] = useState('idle');

    useEffect(() => {
      setStatus('loading');

      fetch('/api/posts')
        .then((res) => res.json())
        .then((data) => {
          setPosts(data);
          setStatus('success');
        })
        .catch(() => setStatus('error'));
    }, []);

    if (status === 'loading') return <p>Loading...</p>;
    if (status === 'error') return <p>Could not load posts.</p>;

    return <ul>{posts.map((post) => <li key={post.id}>{post.title}</li>)}</ul>;
  }
Module 8

Routing

Back to contents

Routing chooses which component to show for a URL, making your app feel like a multi-page experience without full page reloads.

Topics to cover
  • react-router-dom for route definitions and navigation.
  • Link for internal navigation instead of a normal anchor tag.
  • Route parameters such as /products/:id.
  • Protected routes, 404 routes, and lazy-loaded pages.
Mini goals
  • Create Home, About, and Product pages.
  • Read a route parameter with useParams.
  • Redirect unauthenticated users.
Example code
import { BrowserRouter, Link, Navigate, Route, Routes } from 'react-router-dom';

  function ProtectedRoute({ isLoggedIn, children }) {
    return isLoggedIn ? children : <Navigate to="/login" replace />;
  }

  function App() {
    return (
      <BrowserRouter>
        <nav>
          <Link to="/">Home</Link>
          <Link to="/dashboard">Dashboard</Link>
        </nav>

        <Routes>
          <Route path="/" element={<h1>Home</h1>} />
          <Route
            path="/dashboard"
            element={
              <ProtectedRoute isLoggedIn={true}>
                <h1>Dashboard</h1>
              </ProtectedRoute>
            }
          />
        </Routes>
      </BrowserRouter>
    );
  }
Module 9

Forms & Validation

Back to contents

Forms become easier when every input is controlled and validation rules are written clearly.

Key practices
  • Store form values in state.
  • Update fields with onChange.
  • Prevent default form refresh on submit.
  • Show validation feedback next to the related field.
  • Never render raw untrusted HTML into the page.
Validation ideas
  • Required fields
  • Minimum length
  • Email format
  • Password match
Example code
import { useState } from 'react';

  function LoginForm() {
    const [email, setEmail] = useState('');
    const [error, setError] = useState('');

    function handleSubmit(event) {
      event.preventDefault();

      if (!email.includes('@')) {
        setError('Please enter a valid email address.');
        return;
      }

      setError('');
      console.log('Submit form');
    }

    return (
      <form onSubmit={handleSubmit}>
        <input value={email} onChange={(e) => setEmail(e.target.value)} />
        {error && <p>{error}</p>}
        <button type="submit">Login</button>
      </form>
    );
  }
Module 10

Redux

Back to contents

Redux is most useful when many parts of the app need the same state and local state starts getting messy.

Understand this flow
  • UI dispatches an action.
  • A reducer updates state.
  • The store holds the latest state.
  • Components read values with selectors.
  • Redux Toolkit is the modern default choice.
Use Redux when
  • Many distant components need the same data.
  • App state is too large for simple lifting.
  • You want consistent action-driven updates.
Example code
import { createSlice } from '@reduxjs/toolkit';

  const cartSlice = createSlice({
    name: 'cart',
    initialState: { items: [] },
    reducers: {
      addItem(state, action) {
        state.items.push(action.payload);
      }
    }
  });

  export const { addItem } = cartSlice.actions;
  export default cartSlice.reducer;
Rule of thumb: learn Context first for small shared state and Redux Toolkit for larger apps.
Module 11

Authentication

Back to contents

Authentication is about identifying the user safely and controlling access to protected areas of the app.

Core concepts
  • Login, signup, logout, and protected routes.
  • JWT basics and session refresh concepts.
  • HTTP-only cookies are often safer than storing sensitive tokens in localStorage.
  • Auth guards decide whether a route is visible.
Security notes
  • Prefer server-managed auth flows when possible.
  • Avoid exposing sensitive tokens to JavaScript unless you must.
  • Separate auth state from presentational UI.
Example code
import { Navigate } from 'react-router-dom';

  function ProtectedRoute({ user, children }) {
    if (!user) {
      return <Navigate to="/login" replace />;
    }

    return children;
  }
Module 12

Testing Introduction

Back to contents

Good tests focus on what the user sees and does, not the private implementation details inside a component.

Cover these states
  • Success and failure
  • Loading and empty states
  • Clicks, typing, and submit flows
  • Visible feedback after user actions
Testing habit
  • Render the component.
  • Perform a user action.
  • Assert what changed on screen.
Example code
import { render, screen } from '@testing-library/react';
  import userEvent from '@testing-library/user-event';

  test('increments the counter', async () => {
    render(<Counter />);

    await userEvent.click(screen.getByRole('button', { name: /like/i }));

    expect(screen.getByText(/likes: 1/i)).toBeInTheDocument();
  });
Module 13

Deployment

Back to contents

Deployment means packaging your app correctly, checking environment settings, and making sure client-side routes still work after refresh.

Before deploy
  • Run a production build and fix errors.
  • Confirm base paths and asset paths.
  • Check API URLs and environment variables.
  • Make sure the server falls back to index.html for SPA routes.
Final checks
  • App loads after a page refresh.
  • Console has no critical errors.
  • Protected routes still behave correctly.
Example code
import { defineConfig } from 'vite';
  import react from '@vitejs/plugin-react';

  export default defineConfig({
    plugins: [react()],
    base: '/portfolio/'
  });
Module 14

Advanced Topics

Back to contents

These topics become more valuable after the fundamentals feel comfortable. Learn them for leverage, not for memorization.

Explore next
  • Core hooks such as useEffect, useMemo, useCallback, and useContext.
  • Custom hooks for reusable stateful logic.
  • Context API for shared app-level values.
  • Webpack and Babel concepts so tooling feels less mysterious.
  • Animations and transitions for UI polish.
Learning advice
  • Do not start with Saga.
  • Use custom hooks when logic repeats.
  • Reach for memoization only when needed.
Example code
import { useState } from 'react';

  function useCounter(initialValue = 0) {
    const [count, setCount] = useState(initialValue);

    function increment() {
      setCount((current) => current + 1);
    }

    return { count, increment };
  }

Appendix C

Common Mistakes Beginners Make

These mistakes are normal. The goal is to notice them earlier and build the habit of correcting them quickly.

  • Mutating state directly
  • Forgetting keys when rendering lists
  • Using useEffect incorrectly and causing loops
  • Keeping too much state instead of deriving values
  • Using advanced libraries before mastering basics
  • Memorizing syntax without building projects
Fix pattern
  • Replace state instead of mutating it.
  • Give every rendered list item a stable key.
  • Practice tiny projects after each module.
Wrong vs right
// Wrong
  user.name = 'New name';
  setUser(user);

  // Right
  setUser((current) => ({
    ...current,
    name: 'New name'
  }));
Appendix D

Final Advice

You do not need to master everything before building. The fastest path is learning one concept, building one small example, and repeating.

Highest-value path
  • JavaScript fundamentals
  • React basics
  • Hooks
  • Forms
  • API calls
  • Routing
  • Context or Redux Toolkit
  • Testing and deployment
Learning loop
  • Learn one concept.
  • Build one tiny example.
  • Break it and debug it.
  • Repeat with the next concept.
Keep perspective: legacy topics like class components, HOCs, Radium, and Saga are worth recognizing, but they should not slow down your modern React learning path.
